## 페이지네이션

- 특정한 정렬기준 + 필요한 개수의 조건에 맞춰 데이터를 가져오는 것

### 오프셋 기반 페이지네이션

'페이지' 단위로 구분하여 요청 응답하게 만듦

### 커서 기반 페이지네이션

클라이언트가 가져간 마지막 row의 순서상 다음 row들을 n개 요청/응답하게 구현

주로 무한 스크롤이나 더보기를 구현할 때 사용(SNS)
어떤 데이터의 다음에 있다는데에 집중

### 페이지네이션 버튼

페이지네이션 버튼 없이 끊김없는 사용자의 경험을 제공해줌

### 무한 스크롤의 장점

- 사용자의 편의성 : 클릭을 계속 안 해도 됨.
  모바일 최적화 : 터치 환경에서
  자연스러운 탐색, 참여도 향상

### 즉!!! 이래서 Tanstack Query를 무한스크롤에서 사용함

- 무한스크롤을 직접 구현하기 위해서는,
  상태를 직접 구현해야됨

const [data,setData] = useState([]);
-> 그래서 이렇게 해서 상태값을 기억하고 상태가 변화하면 바꿔주는 기능을 한다.

- 자동캐싱 : 한번 불러온 데이터를 기억한다.
- 로딩/에러 상태 : 알아서 관리해줌
- 리패칭 : 데이터가 오래되면 알아서 새로고침
- 무한 스크롤 전용 훅 : useInfiniteQuery 도구 이용가능
